## Overview

This document specifies the **creative core** of Computational Serendipity — the system that transforms a mundane Context Snapshot into a surprising, insightful interjection. This is where the "magic" happens: the hidden causal chain that produces the appearance of genuine creativity.

The pipeline: **Seed Topics → Branching Association Tree → Web Search → Interest Scoring → Bridge Building → Interjection**

---

## 1. Association Tree Generator

### 1.1 What Is an Association?

An association is a conceptual leap from one topic to a related (but different) topic. The key: each hop should move into **adjacent but distinct** conceptual territory.

**Good associations (cross-domain):**
- "Concrete foundations" → "Foundation of a philosophical argument" (construction → philosophy)
- "Debugging code" → "Debugging a relationship" (programming → psychology)
- "Load-bearing walls" → "Load-bearing assumptions in science" (architecture → epistemology)

**Bad associations (same-domain, too obvious):**
- "Concrete foundations" → "Concrete mix ratios" (construction → construction)
- "Debugging code" → "Linting code" (programming → programming)

### 1.2 Tree Structure

```
                         [Seed Topic from Context]
                         /          |          \
                    [Hop 1a]    [Hop 1b]    [Hop 1c]
                    /  |  \     /  |  \     /  |  \
                 [2a][2b][2c] [2d][2e][2f] [2g][2h][2i]
                  ... continuing to depth 4-7 ...
```

**Configuration:**

| Parameter | Value | Notes |
|---|---|---|
| Branching factor | 3 | Ternary tree at each node |
| Minimum depth | 4 | Shortest path to a candidate |
| Maximum depth | 7 | Longest path before noise dominates |
| Required domain crossings | ≥ 1 | At least one hop must change fields |
| Seeds per heartbeat | 1–3 | Extracted from Context Snapshot by novelty weight |

### 1.3 Hop Generation — LLM Prompt Strategy

Each hop is generated by prompting the LLM with a specific instruction. The prompt evolves by depth:

**Hop 1 (Grounded):**
```
Given the topic "{seed_topic}" which comes from the user's current context,
generate 3 creative lateral associations. Each association should:
- Connect to the seed topic in a non-obvious way
- Ideally cross into a different domain or field
- Be a real concept, not abstract nonsense

Return as a JSON array of 3 objects:
[{"topic": "...", "domain": "...", "connection_reason": "..."}]
```

**Hops 2-4 (Exploring):**
```
You are building a chain of creative associations. The chain so far is:
{chain_so_far}

From the current topic "{current_topic}" (domain: {current_domain}),
generate 3 lateral associations that:
- Move AWAY from the original seed topic (do NOT circle back)
- Cross into a DIFFERENT domain than the current one if possible
- Are surprising but defensible (not random)

Return as a JSON array of 3 objects:
[{"topic": "...", "domain": "...", "connection_reason": "..."}]
```

**Hops 5-7 (Deep exploration, optional):**
```
Same as above, but with additional instruction:
- You are now far from the origin. Prioritize the MOST surprising and 
  specific association. Prefer concrete facts over abstract concepts.
- If you can't find a genuinely interesting next hop, return fewer 
  than 3 — quality over quantity at this depth.
```

### 1.4 Variable Depth — When to Stop

Not all branches need to go to the same depth. A branch can terminate early if:

1. **Large semantic jump detected:** If a single hop covers exceptional semantic distance (cosine distance > 0.7 between embeddings), it may have already reached "interesting territory" — no need to keep going.
2. **Quality degradation:** If the LLM returns associations that are vague or abstract ("the nature of existence"), the branch is getting unproductive. Prune it.
3. **Domain crossing achieved:** If the minimum domain crossing requirement is met and the semantic distance is sufficient, the branch can stop.

```python
def should_continue_branch(chain: list[AssociationNode], min_depth: int) -> bool:
    if len(chain) < min_depth:
        return True  # haven't reached minimum yet
    
    total_semantic_distance = compute_chain_distance(chain)
    domain_crossings = count_domain_crossings(chain)
    
    if total_semantic_distance > 0.8 and domain_crossings >= 1:
        return False  # far enough and crossed domains — good endpoint
    
    if len(chain) >= MAX_DEPTH:
        return False  # hit the ceiling
    
    return True  # keep going
```

### 1.5 Domain Classification

Each topic is classified into a broad domain to track domain crossings:

**Domain categories (expandable):**
- Science & Technology
- Psychology & Behavior
- Art & Design
- History & Culture
- Nature & Biology
- Philosophy & Ethics
- Economics & Business
- Mathematics & Logic
- Language & Literature
- Sports & Games
- Music & Sound
- Food & Cooking
- Architecture & Space
- Politics & Society
- Health & Medicine

The LLM assigns a domain label at each hop. A "domain crossing" is any hop where the domain changes from the previous node.

## 2. Web Search Module

### 2.1 Purpose

The association tree generates **conceptual endpoints**. The web search grounds those endpoints in **real facts, statistics, and stories** — turning abstract associations into concrete, shareable insights.

### 2.2 Process

```
Top 3-5 candidate endpoints (from preliminary scoring)
    │
    ▼
┌─────────────────────────┐
│  Query Constructor       │  → Turn the topic into a search query
│  "astronaut psychology   │     e.g. "astronaut psychology team
│   under pressure"        │      dynamics isolation research"
└───────────┬─────────────┘
            │
            ▼
┌─────────────────────────┐
│  Web Search API          │  → Execute search
│  (Tavily/SerpAPI/Brave)  │     Return top 3-5 results per query
└───────────┬─────────────┘
            │
            ▼
┌─────────────────────────┐
│  Result Extractor        │  → Pull key facts, stats, quotes
│  (LLM summarization)    │     from search results
└───────────┬─────────────┘
            │
            ▼
  Output: SearchResult(topic, facts[], source_urls[], summary)
```

### 2.3 Query Construction

The LLM constructs a search query from the association endpoint:

```
Given this creative association endpoint: "{endpoint_topic}"
Which arrived through this chain: {chain_summary}

Construct a web search query that would find:
- Surprising facts or statistics about this topic
- Real-world examples or stories
- Scientific research if applicable

The query should be specific enough to return interesting results,
not a generic overview. Aim for the kind of search a curious 
person would do at 2 AM when they fell down a Wikipedia rabbit hole.

Return just the search query string.
```

### 2.4 Result Quality Filter

Not all search results are useful. Filter for:
- **Factual content** (not ads, not listicles)
- **Specific details** (numbers, names, dates — not vague summaries)
- **Surprising content** (if the LLM already "knew" this from training data, it's less interesting)

## 3. Interest Scorer

### 3.1 Scoring Formula

Each candidate endpoint (with its chain and search results) is scored:

```
interest_score = (semantic_distance × 0.30)
               + (domain_crossings  × 0.25)
               + (surprise          × 0.20)
               + (bridgeability     × 0.15)
               + (novelty           × 0.10)
```

### 3.2 Metric Computation

#### Semantic Distance (weight: 0.30)

How far is the endpoint from the original seed topic?

```python
def compute_semantic_distance(seed_embedding, endpoint_embedding) -> float:
    """
    Cosine distance (not similarity) between seed and endpoint.
    Range: 0.0 (identical) to 1.0 (maximally distant)
    """
    similarity = cosine_similarity(seed_embedding, endpoint_embedding)
    return 1.0 - similarity
```

**Normalization:** Raw cosine distances in embedding space rarely exceed 0.8 for meaningful concepts (truly orthogonal concepts). Normalize to 0–1 range based on observed distribution.

#### Domain Crossings (weight: 0.25)

How many times did the chain jump between fields?

```python
def compute_domain_crossing_score(chain: list[AssociationNode]) -> float:
    """
    Count domain boundary crossings, normalized.
    1 crossing = 0.4, 2 crossings = 0.7, 3+ crossings = 1.0
    """
    crossings = 0
    for i in range(1, len(chain)):
        if chain[i].domain != chain[i-1].domain:
            crossings += 1
    
    scoring_map = {0: 0.0, 1: 0.4, 2: 0.7}
    return scoring_map.get(crossings, 1.0)  # 3+ = 1.0
```

#### Surprise (weight: 0.20)

Would this association be expected or unexpected?

```python
def compute_surprise(seed_topic: str, endpoint_topic: str, llm) -> float:
    """
    Ask the LLM: how surprising is this connection?
    Uses the LLM's own probability estimation as a proxy.
    """
    prompt = f"""
    On a scale of 0.0 to 1.0, how surprising is it to connect 
    "{seed_topic}" to "{endpoint_topic}"?
    
    0.0 = completely obvious (anyone would make this connection)
    0.5 = moderately surprising (clever but not shocking)  
    1.0 = highly surprising (genuinely unexpected connection)
    
    Return only the number.
    """
    return float(llm.generate(prompt))
```

#### Bridgeability (weight: 0.15)

Can the LLM construct a compelling narrative connecting the endpoint back to the conversation? This replaced "relevance snap-back" because snap-back penalized bold associations.

```python
def compute_bridgeability(
    chain: list[AssociationNode], 
    context: ContextSnapshot,
    search_results: SearchResult,
    llm
) -> float:
    """
    Can a compelling story be told connecting this back to the user's world?
    The bridge IS the insight.
    """
    prompt = f"""
    A creative association chain went from "{chain[0].topic}" to 
    "{chain[-1].topic}" through these steps: {chain_summary}.
    
    The user's current context: {context.summary}
    
    Relevant facts found: {search_results.summary}
    
    Can you construct a compelling, interesting reason why this 
    endpoint matters in the context of what the user is doing?
    
    Rate your confidence in the bridge on a scale of 0.0 to 1.0:
    0.0 = no meaningful connection possible
    0.5 = there's a connection but it's a stretch
    1.0 = this is a genuinely illuminating connection
    
    Return the score and a one-sentence bridge.
    """
    result = llm.generate(prompt)
    return result.score, result.bridge_sentence
```

#### Novelty (weight: 0.10)

Has this topic or connection been raised before?

```python
def compute_novelty(endpoint_embedding, past_interjections: list) -> float:
    """
    Compare this endpoint to all previous interjections.
    High novelty = we've never talked about anything like this.
    """
    if not past_interjections:
        return 1.0
    
    similarities = [
        cosine_similarity(endpoint_embedding, past.embedding)
        for past in past_interjections
    ]
    max_similarity = max(similarities)
    return 1.0 - max_similarity
```

### 3.3 Scoring Thresholds

| Score Range | Action |
|---|---|
| **0.65 – 1.0** | ✅ Proceed to Bridge Builder → deliver interjection |
| **0.40 – 0.64** | 🧪 Save to Incubation Queue → re-evaluate on future heartbeats |
| **0.00 – 0.39** | ❌ Discard — not interesting enough |

### 3.4 The Incubation Queue

Ideas that aren't interesting enough *right now* but might become interesting later.

```python
@dataclass
class IncubatedIdea:
    endpoint: AssociationNode
    chain: list[AssociationNode]
    search_results: SearchResult
    original_score: float
    original_context: ContextSnapshot
    created_at: datetime
    times_reevaluated: int = 0
    max_reevaluations: int = 5      # after 5 tries, expire
```

On each heartbeat, after processing new associations, also re-score 1-2 incubated ideas against the **current** context. If context has shifted enough, a previously mediocre association might now score above threshold.

**Expiration:** Ideas are removed from incubation after 5 re-evaluations or 24 hours, whichever comes first.

## 4. Bridge Builder

### 4.1 Purpose

Transform the raw association chain + search results into a **natural, conversational interjection** that feels like a friend sharing an interesting thought.

### 4.2 Prompt Template

```
You are a curious, creative friend who has been hanging out with the user.
You just had an interesting thought and want to share it naturally.

CONTEXT (what the user is doing right now):
{context_snapshot.summary}

YOUR THOUGHT PROCESS (do NOT share this chain — it's internal):
{association_chain_summary}

WHAT YOU FOUND (interesting facts from your research):
{search_results.summary}

THE BRIDGE (why this connects to what the user is doing):
{bridge_sentence}

Now share your thought with the user. Rules:
- Sound like a friend, not a professor or assistant
- Be genuinely enthusiastic but not over-the-top
- Keep it to 2-3 sentences max — like tapping someone on the shoulder
- Do NOT explain your association chain or say "I was thinking about X 
  which led me to Y which led me to Z"
- Just share the interesting insight naturally
- It's okay if the connection to what the user is doing is loose — 
  friends go on tangents
- Include a specific fact, number, or detail — not vague generalities
```

### 4.3 Interjection Style Examples

**Good interjection:**
> "Oh hey — did you know that NASA actually studied how construction workers communicate under stress when they were designing protocols for ISS repair crews? Turns out the hand signals are almost identical. Wild, right?"

**Bad interjection (too academic):**
> "I've been analyzing the cross-domain parallels between construction site communication protocols and aerospace engineering team dynamics, and found that NASA's research in this area yielded significant findings regarding non-verbal communication under high-stress conditions."

**Bad interjection (too vague):**
> "I was thinking about space and construction and how they're kind of related in interesting ways."

**Bad interjection (chain exposed):**
> "So I started thinking about contractors, which made me think about building, which made me think about rockets, which made me think about space construction, and then I found this article about NASA..."

### 4.4 Tone Calibration

The interjection tone should adapt to context:

| Context | Tone |
|---|---|
| User is focused on deep work | Brief, low-key: "Quick thought — ..." |
| User seems relaxed/browsing | Conversational, exploratory |
| User just expressed frustration | Lighter, fun fact to break tension |
| Late at night | More reflective, philosophical |

## 5. Full Pipeline Timing

```
Heartbeat fires (T=0)
    │
    ├─ Input capture & novelty weighting      T+0 to T+5s
    │
    ├─ Context assembly & seed extraction     T+5s to T+8s
    │
    ├─ Association tree generation            T+8s to T+20s
    │  (3-5 LLM calls for branching)
    │
    ├─ Preliminary scoring                    T+20s to T+22s
    │  (embedding comparisons, fast metrics)
    │
    ├─ Web search (top 3-5 candidates)        T+22s to T+30s
    │
    ├─ Final scoring (with bridgeability)     T+30s to T+35s
    │
    ├─ Bridge building (interjection text)    T+35s to T+40s
    │
    └─ Delivery                               T+40s
    
Total: ~40 seconds from heartbeat to interjection
```

## 6. Optimization Strategies

### 6.1 Reduce LLM Calls

The association tree with branching factor 3 and depth 5 would naively require many LLM calls. Strategies:

- **Batch hop generation:** Ask the LLM to generate all 3 branches in one call (already in the prompts above)
- **Depth-first pruning:** Score after each level, prune low-scoring branches before expanding further
- **Cache associations:** If the same topic appears in multiple chains, reuse its branches
- **Embed locally:** Use local embedding models (sentence-transformers) for all similarity computations — only use the LLM for generation tasks

### 6.2 Depth-First Pruning (Recommended)

Instead of generating the full tree and then scoring, prune as you go:

```
Level 1: Generate 3 branches → quick-score all 3 → keep top 2
Level 2: Generate 3 branches each (6 total) → quick-score → keep top 3
Level 3: Generate 3 branches each (9 total) → quick-score → keep top 3
...
```

This reduces the candidate pool from 243 to roughly 20-30, cutting LLM calls significantly while keeping the best paths.

### 6.3 LLM Call Budget Per Heartbeat

| Step | Estimated LLM Calls | Notes |
|---|---|---|
| Image description | 0–1 | Only if vision novelty > 0.3 |
| Seed topic extraction | 1 | From context snapshot |
| Association generation | 5–8 | With pruning (vs 15+ without) |
| Search query construction | 3–5 | One per top candidate |
| Surprise scoring | 3–5 | One per top candidate |
| Bridgeability scoring | 3–5 | One per top candidate |
| Bridge building | 1 | Final interjection text |
| **Total per heartbeat** | **~16–26 calls** | **Avg ~20 calls per cycle** |

At one heartbeat every ~5 minutes (average), that's roughly **240 LLM calls per hour** during active use. Budget planning should account for this.

## 7. Data Structures

```python
@dataclass
class AssociationNode:
    topic: str
    domain: str
    connection_reason: str
    embedding: np.ndarray
```

```python
@dataclass
class AssociationChain:
    seed_topic: str
    seed_context: ContextSnapshot
    nodes: list[AssociationNode]    # ordered root to leaf

    domain_crossings: int
    search_results: SearchResult | None = None
    interest_score: float | None = None
    bridge_sentence: str | None = None
```

### Interjection

```python
@dataclass
class Interjection:
```
